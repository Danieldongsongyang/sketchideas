# PRD｜Sketch Ideas 视觉老虎机站（重写版）

## 1. 产品概述
**一句话**：用“视觉老虎机”形式快速触发灵感，让用户在几秒内获得可临摹的线稿参考图，并通过筛选与连续刷新形成高停留与高复访。

**核心假设**：用户搜“sketch ideas”不是来阅读列表，而是想被触发冲动；快速、稳定、可连续刷的视觉刺激能显著提升停留与转化。

## 2. 目标与成功标准（MVP 30 天）
- 日均停留：3–5 分钟
- 单次会话 Spin 次数：≥ 15
- 平均加载时间：≤ 1.5s
- SEO 页面收录率：≥ 30%

## 3. 目标用户与场景
- **Warm-up 初学者**：不想思考，只想快速热身
- **Sketch 进阶者**：练结构与构图，需要有空间关系
- **Challenge 挑战者**：练叙事、光线与镜头语言

## 4. 核心价值
- 视觉老虎机式“触发冲动灵感”
- 线稿/结构参考图降低 AI 味，提高信任
- 预生成图池保证速度与质量

## 5. 核心用户路径（MVP）
1) 进入页面 → 选择难度/类别 → 点击 Spin
2) 看到线稿 + prompt → 继续 Next
3) 复制/下载/分享 → 继续刷新或离开

## 6. 页面架构与功能分配

基于 ShipAny Template Three 的页面结构，将功能分配到以下页面：

### 6.1 Features 页面 - 视觉老虎机核心功能
**定位**：产品核心功能展示页，包含完整的 Spin 视觉老虎机体验

**核心功能**：
1. **Spin（视觉老虎机）**
   - 输出：文字提示 + 线稿参考图
   - 逻辑：从预生成库抽取
   - 速度：≤ 1s 出结果
   - 动效：老虎机式快速切图/滚动（0.5s 动画）


2. **类别筛选（Category）**
   - **Random**：全分类随机抽取
   - **Animals**：动物素描
   - **Anime_Bleach**：死神动漫风格
   - **Anime_Characters**：动漫角色
   - **Anime_Characters_Ancient_Style**：古风动漫角色
   - **Anime_Characters_Full_Body**：动漫全身角色
   - **Anime_Chibi**：Q版动漫
   - **Anime_Naruto**：火影忍者动漫风格
   - **Dynamic_Sketches**：动态速写
   - **Flowers**：花卉素描
   - **Flowers_and_Birds**：花鸟素描
   - **Flowers_Plants_and_Trees**：花草树木
   - **Food**：食物素描
   - **Forest**：森林场景
   - **Kids_Cartoons**：儿童卡通
   - **Objects_and_Scenes**：物品与场景
   - **Ocean**：海洋主题
   - **Pen_Control_Practice**：控笔练习
   - **Toddler_Cartoons**：幼儿卡通


3. **下载/复制**
   - 下载线稿图（WebP）
   - 一键复制 prompt（Tooltip 教学）


**页面结构**：
- 顶部：产品介绍与核心价值主张
- 中部：完整的 Spin 视觉老虎机界面（左侧筛选 + 中间老虎机显示区）
- 底部：功能特点说明与使用指南

### 6.2 Showcases 页面 - 素材商城
**定位**：付费/免费线稿图集商城，用户可浏览、筛选、购买或下载图集

**核心功能**：
1. **图集展示**
   - 按分类展示图集（Category 筛选）
   - 每个图集包含：封面图、标题、描述、素材数量、价格/免费标识
   - 支持网格/列表两种展示模式

2. **筛选与搜索**
   - 分类筛选（与 Features 页面保持一致的 19 个分类）
   - 价格筛选（免费 / 付费）
   - 关键词搜索

3. **图集详情**
   - 图集预览（显示部分缩略图）
   - 完整描述与适用场景
   - 下载/购买按钮
   - 用户评价（可选）

4. **购物车与结账**
   - 添加到购物车
   - 集成 Stripe/PayPal 支付
   - 免费图集直接下载
   - 付费图集支付后下载

5. **用户中心**（可选，MVP 后）
   - 我的购买记录
   - 收藏的图集
   - 下载历史

**页面结构**：
- 顶部：搜索栏与筛选器
- 中部：图集展示网格
- 底部：商城说明与版权声明

**变现策略**：
- 免费图集：引流，获取用户邮箱
- 付费图集：单次购买或订阅制
- 捆绑包：相关分类打包优惠

### 6.3 AI 页面 - 图片转线稿功能
**定位**：让用户上传任意图片，一键转换为线稿参考图，提供从照片到临摹素材的快速转换

**核心功能**：
1. **图片上传**
   - 支持拖拽上传
   - 支持点击选择文件
   - 支持常见图片格式（JPG、PNG、WebP）
   - 文件大小限制（建议 ≤ 10MB）

2. **一键转换**
   - 单一按钮：上传后自动转换
   - 转换进度提示
   - 转换时间 ≤ 5s

3. **结果预览与对比**
   - 左右分屏对比：原图 vs 线稿
   - 滑动对比条（可选）
   - 全屏预览模式

4. **结果处理**
   - 下载线稿图（WebP/PNG 格式）
   - 重新转换

5. **额度系统**
   - 免费用户每日 3 次转换
   - 付费用户无限转换或购买额度包
   - 注册用户每日 5 次转换

**页面结构**：
```
┌─────────────────────────────────────────┐
│          AI 图片转线稿                  │
├─────────────────────────────────────────┤
│  ┌───────────┐    ┌───────────┐         │
│  │  上传区   │ →  │  结果区   │         │
│  │  拖拽或   │    │  原图|线稿│         │
│  │  点击上传 │    │  对比显示 │         │
│  └───────────┘    └───────────┘         │
│                                         │
│  操作按钮：[重新转换] [下载线稿]       │
│                                         │
│  剩余额度：今日 2/3 次                 │
└─────────────────────────────────────────┘
```

**技术实现**：
- AI 扩展：集成 Replicate / Fal 的图片转线稿模型
- 推荐模型：ControlNet Scribble / Canny
- 队列管理：避免超时，支持大批量转换
- CDN 存储：转换结果缓存 24 小时

## 9. 变现策略（更新）

基于三页架构的多元化变现模式：

### 9.1 Features 页面（引流与免费体验）
- **目标**：获取流量，建立品牌认知
- **变现方式**：
  - AdSense 广告（页面顶部与底部）
  - Affiliate 链接（绘图工具、书籍推荐）
  - 邮箱收集（Spin 结果需注册后下载）

### 9.2 Showcases 页面（素材商城 - 主要收入）
- **目标**：直接变现，付费图集销售
- **变现方式**：
  - 单次购买：单个图集 $2.99 - $9.99
  - 捆绑包：相关分类打包 $19.99 - $49.99
  - 订阅制：月度 $9.99 / 年度 $79.99（无限下载）
  - 免费图集：邮箱交换（获取用户邮箱）

### 9.3 AI 页面（高级功能与增值服务）
- **目标**：提供图片转线稿功能，额度付费
- **变现方式**：
  - 免费额度：每日 3 次转换
  - 额度包：$4.99 / 30 次，$9.99 / 100 次
  - 无限订阅：月度 $9.99，年度 $79.99
  - 高级功能：批量转换、高分辨率输出（仅订阅用户）

### 9.4 三页协同变现流程
```
用户进入 Features 页面
    ↓
免费 Spin 体验 → 收集邮箱
    ↓
    ├─→ 需要更多素材 → Showcases 页面
    │                    ↓
```

## 11. 技术需求（更新）

### 11.1 基础架构
- 预生成图库（元数据：difficulty / category / tags / style）
- 抽取逻辑：可控随机 + 去重
- CDN 静态加速
- 可扩展数据库结构（预留用户扩展）

### 11.2 三页技术实现

**Features 页面**：
- 前端：React + Tailwind CSS（复用 sketch-ideas-slot.html 设计）
- 后端：预生成图库 API（从 CDN 加载）
- 动效：Framer Motion / CSS 动画
- 状态管理：Zustand / React Context

**Showcases 页面**：
- 前端：网格布局 + 筛选组件
- 后端：图集数据 API
- 支付：Stripe / PayPal 集成（使用模板 Payment 扩展）
- 存储：Cloudflare R2（使用模板 Storage 扩展）

**AI 页面**：
- 前端：拖拽上传组件 + 对比预览组件
- 后端：图片转换队列 + 历史记录管理
- AI 集成：使用模板 AI 扩展（Replicate / Fal ControlNet）
- 额度系统：用户数据库 + Redis 限流

### 11.3 性能优化
- 强制 WebP 图片格式
- Lazy Loading 图片懒加载
- 图库分页加载（每页 20-30 个图集）
- 图片转换结果缓存（避免重复转换，24 小时）
- 目标加载时间 ≤ 1.5s

### 11.4 数据库 Schema
```sql
-- 预生成图库表
sketch_images (
  id, url, prompt, difficulty, category,
  tags, created_at, is_active
)

-- 图集表（Showcases）
collections (
  id, title, description, cover_image,
  category, difficulty, image_count,
  price, is_free, created_at
)

-- 图集图片关系表
collection_images (
  collection_id, image_id, order_index
)

-- 用户额度表（AI 页面）
user_credits (
  user_id, daily_limit, used_today,
  last_reset_date, total_purchased
)

-- 图片转换历史表（AI 页面）
image_conversions (
  id, user_id, original_image_url, result_image_url,
  parameters, created_at
)
```

## 13. 风险与规避

| 风险 | 规避措施 |
|------|----------|
| AI 味过强 | 统一线稿风格 + 结构参考导向 + 临摹点标注 |
| 随机疲劳 | 类似更多/更随机 |
| SEO 薄页 | 动态标题 + 面包屑 + FAQ + 文本厚度 + 精选图 |
| 版权问题 | 自生成图库 + AI 生成声明 + 仅供练习参考 |
| 风格不一致 | 统一线稿风格规范 |
| AI 转换成本过高 | 限制免费额度 + 缓存转换结果 + 队列管理 |
| 支付流失 | 免费图集引流 + 多支付方式 + 订阅制 |
